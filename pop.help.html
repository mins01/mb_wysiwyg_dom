HELP 도움말
소개
  • iframe를 사용한 위지윅에디터입니다.
  • Full JavaScript로 되어있습니다.(다만 이미지 업로드만 해당 언어의 업로드 기능을 사용하였습니다)
  • 툴바,아이콘,색상테이블창 등등을 간단한 설정 변경으로 설정하실 수 있습니다.
  • FF와 MSIE, Opera에서 최적화되어있습니다.
    (Opera 에서는 동작은 하지만 불안정합니다. 비추천)
  • Chrome, Safari 지원됩니다.(20090725)
기본사용법

  1. <link href="./mb_wysiwyg.css" rel="stylesheet" type="text/css"> // 위지윅 스타일 시트파일
    <script src="./set.toolbar.js" charset='UTF-8'></script> // 툴바 설정파일 JS파일 (내용을 임의로 수정해도 됩니다.없으면 기본값으로 동작)
    <script src="./mb_wysiwyg.js" charset='UTF-8'></script> // 위지윅 JS파일
    세 파일을 링크시킵니다.(여러개의 위지윅 에디터를 사용해도 한번만 링크하시면 됩니다.)

  2. var wysiwyg = new mb_wysiwyg(대상,너비,높이,[기본내용:옵션],[제목:옵션]);
    // 위지윅에디터 생성 (기본내용이 없으면 대상의 value를 가져옴)
    wysiwyg.path="."; // mb_wysiwyg.js 파일이 있는 경로를 적어줘야한다.(ver 20070724 부턴 안적어도 됨, 자동으로 처리)
    wysiwyg.icon_pack="black_16px"; // 아이콘팩 설정 (작은 검은색 아이콘)
    //or wysiwyg.icon_pack="wr_24px"; // 아이콘팩 설정 (큰 흰색 아이콘)
    wysiwyg.c_table_type = true; // 컬러픽커 설정 : true:레이어, false:팝업
    wysiwyg.mk_wysiwyg(); //위지윅 에디터 동작
  3. wysiwyg.isXhtml = true; //XHTML 형식으로 태그가 수정됨

  4. form의 obsubmit 이벤트에 obsubmit="wysiwyg.submit(); etc();"처럼 등록
    (wysiwyg.submit();를 동작안시키면 textarea안의 내용이 보이는 것과 다를 수 있습니다.)
    (여러개의 위지윅을 사용하셔도 하나만 등록시키시면 됩니다.)

  5. 다중 위지윅 에디터를 사용할 경우
    var wysiwyg2 = new mb_wysiwyg(대상,너비,높이,[기본내용:옵션],[제목:옵션]);
    (변수의 이름만 다르게 하시면 됩니다. 그 변수 이름을 기준으로 설정하시면 됩니다.)
  • 추가설정
    이 위지윅 에디터는 PHP에서 사용시 이미지 업로드를 지원합니다.
    설정된 폴더에서 upload 라는 하위폴더의 권한을 707(or 777)로 바꿔주셔야합니다.
    업로드가 지원안되거나 사용 안하실 때에는 pop.insertImage.html 파일에서 use_upload = false로 해주셔야 이미지 업로드 부분이 안보입니다.

지원 환경
  • MSIE 6.0+
  • FireFox 1.5+ (Genko계열)
  • Opera 9.0+
  • Chrom 2.0.X+
  • Safari 3.1.1+(WIN용만 실험됨, MAC은 실험 안되었음)
  • JavaScript가 동작해야합니다. JSDOM이 지원되야합니다.
  • Iframe이 지원되야합니다.
  • 이미지 업로드는 현재(20060920) PHP(4.1+)에서만 정상 작동합니다.
지원 안되는 브라우저
  • JavaScript 가 지원 안되는 브라우저 , DOM 지원이 안되는 Javascript
  • Iframe 의 디자인 모드가 지원 안되는 브라우저
알려진 버그
  • Opera:
    이미지와 테이블을 삽입시 삽입이 되지 않고 오류가 발생합니다.
    (20060920 해결)
  • Opera:
    이미지와 텍스트를 동시 선택(block) 후 폰트를 바꾸면 이미지가 복제됨
    (해결된 것 같음 : DOM버전)
  • FF:
    웹(위지윅)에디터가 처음 동작시 아무 내용도 안들어있다면, 에디터에 포커스가 가지 않는 현상이 있음(글을 적을 수 있는 커서가 보이지 않음)
    -> Design 버튼을 한번 클릭 후 다시 적기를 시도하면 됨
    -> 해결됨 (v20061206);
  • FF 2:
    테이블 삽입창, 이미지 삽입창 같은 경우 Modeless창으로 안 뜸(FF 1.5에서는 Modeless 창으로 뜸)
    ->FF 2의 버그로 예상, 아직까지 해결법이 없음, 일반 팝업창으로 뜨지만, 사용에는 아무런 문제가 없음
  • FF:
    미디어 삽입시 개체가 삽입 안되고 태그가 문자열로 삽입됨
  • IE:
    위지윅창에서 뒤로가기를 하면 현재의 창이 나타남
